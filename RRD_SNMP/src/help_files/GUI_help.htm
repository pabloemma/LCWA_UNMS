The LCWA UNMS GUI

 

This is the help file for the UNMS GUI

 

General: In order to connect to the UNMS system, you have to have a VPN connection into the 172 network. The program won’t be able to run without this connection established.

The main control is through the menu bar:



The first thing you need to do is login: Go to action and click on login.

You will see the login screen:













If you want to connect to a different UNMS server, just give the IP address. Now that you are logged in you have a few choices. Most of the choices you want are under the Services tab:





Probably the most useful one is Get SiteID. Everything in the communication is built on the SiteID.

Click that one and enter the name of the place you want to look at:









This would allow you to look at all the properties of the madre-de-dios radio. If you choose an AP like RidgeRoad, you could then get more info on either RidgeRoad itself or its clients.

The most useful service is the “Get SiteStatistics”. It lets you plot the traffic between the site you have chosen and its parent site for hour, day or month.











This is what the site statistic will look like for an hour (Clearly you can see the speedtest every 10’):













Or for a day:











The output is controlled by clicking on output file in “action” . If you click this menuitem, it will ask you for an output file and directory, and all subsequent output will be in this file. However if you click on “Get AllAPs” or “Get AllSSIDs” in services , it will automatically redirect the output to a file, which will reside in ~/UNMS/output. So it is a good idea to create this directory if you want to use this feature.

There is also a nonUNMS tab, which lets you do an iperf3 test between two sites. For this you need to be connected to the 172 network.
